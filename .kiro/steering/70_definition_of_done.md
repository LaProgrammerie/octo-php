# DoD

Une tâche est “done” si :
- tests ajoutés (unit/integration selon zone)
- observabilité : traces + métriques pertinentes
- timeouts/limits/cancel appliqués
- docs mises à jour (README/ADR si structurant)
- bench exécuté si perf impactée (comparatif avant/après)
